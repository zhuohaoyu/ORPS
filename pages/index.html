The Challenge in Code Generation Rewards

Current reward models for code generation face a fundamental trade-off between process and outcome rewards, limiting their effectiveness in training robust code generation models.

Process Rewards

Evaluate reasoning steps during generation

‚úÖ Advantages:

  • ‚Ä¢ Provide dense supervision signals
  • ‚Ä¢ Guide intermediate reasoning steps
  • ‚Ä¢ Help with interpretability

‚ùå Limitations:

  • ‚Ä¢ Expensive to annotate step-by-step reasoning
  • ‚Ä¢ May not capture final code correctness
  • ‚Ä¢ Can overfit to specific reasoning patterns

Outcome Rewards

Evaluate final results and correctness

‚úÖ Advantages:

  • ‚Ä¢ Easy to obtain through code execution
  • ‚Ä¢ Directly measure final correctness
  • ‚Ä¢ Objective and unbiased evaluation

‚ùå Limitations:

  • ‚Ä¢ Sparse reward signals
  • ‚Ä¢ No guidance on reasoning process
  • ‚Ä¢ May lead to reward hacking

The Fundamental Challenge

Existing approaches typically use either process rewards or outcome rewards, missing the opportunity to leverage both types of supervision effectively.

Suboptimal Training

Using only one type of reward leads to incomplete learning and suboptimal code generation performance.

Reward Signal Mismatch

Process and outcome rewards often provide conflicting signals, leading to training instability.

Limited Scalability

Manual annotation of process rewards is expensive and doesn't scale to diverse coding scenarios.

Our Solution: ORPS

Outcome and Reasoning Process Supervision unifies both reward types through execution-based reasoning, achieving the best of both worlds.

üîÑ

1. Execution-Based Process Supervision

Instead of relying on expensive manual annotations, ORPS automatically generates process supervision by executing code step-by-step and analyzing the intermediate states.

This approach provides dense, accurate feedback about the reasoning process while maintaining scalability and objectivity.

üéØ

2. Unified Reward Framework

ORPS seamlessly combines process and outcome rewards into a single, coherent training objective that optimizes both reasoning quality and final correctness.

This unified approach eliminates conflicts between different reward signals and leads to more stable and effective training.

üí° Example: Training with ORPS

from orps import ORPSTrainer, ExecutionAnalyzer

# Initialize ORPS trainer
trainer = ORPSTrainer(
    model_name="codellama-7b",
    process_weight=0.3,      # Weight for process rewards
    outcome_weight=0.7,      # Weight for outcome rewards
    execution_timeout=10     # Timeout for code execution
)

# Set up execution analyzer
analyzer = ExecutionAnalyzer(
    enable_step_tracking=True,
    capture_intermediate_states=True
)

# Training loop
for batch in dataloader:
    # Generate code with reasoning
    generated_code = trainer.generate(batch.problems)
    
    # Analyze execution and get rewards
    process_rewards = analyzer.analyze_process(generated_code)
    outcome_rewards = analyzer.analyze_outcomes(generated_code, batch.test_cases)
    
    # Unified training step
    loss = trainer.compute_loss(process_rewards, outcome_rewards)
    trainer.optimize(loss)

print("Training completed with ORPS!")

Key Features & Innovations

ORPS introduces several novel techniques for effective code generation training

Execution-Based Process Supervision

Automatically generate process rewards through step-by-step code execution and state analysis.

Unified Reward Framework

Seamlessly combine process and outcome rewards into a single, coherent training objective.

Scalable Supervision

No manual annotation required - supervision scales automatically with execution environments.

Reasoning Interpretability

Models learn to generate code with explicit reasoning that can be verified through execution.

Multi-Language Support

Framework supports multiple programming languages with configurable execution environments.

Robust Training

Stable training dynamics through balanced process and outcome supervision signals.
