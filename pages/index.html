The Challenge: Complex Code Generation

Large Language Models excel at basic coding tasks but struggle with complex programming challenges that require multi-step algorithmic reasoning and strategic thinking.

Current Supervision Paradigms Fall Short

‚ùå Outcome Supervision:

Only evaluates final code quality (pass/fail on tests). Provides no guidance on how to improve or what went wrong. Fails for complex tasks requiring coordinated intermediate steps.

‚ùå Process Supervision:

Relies on costly Process Reward Models (PRMs) that require expensive training data and suffer from reward hallucination and hacking. Math-focused PRMs don't suit programming's structured logic.

Challenge #1: Local Repair vs Strategic Thinking

Existing methods focus on fixing code line-by-line, missing opportunities for better algorithmic approaches.

Challenge #2: Expensive Process Supervision

Process Reward Models require costly training and suffer from hallucination and reward hacking.

‚ö†Ô∏è PRM Problems:

Trained PRMs misjudge code quality, learn superficial patterns, and require expensive human annotations

‚ùå PRM Limitations

‚Ä¢ Reward Hallucination: Assigns high scores to buggy code
‚Ä¢ Superficial Patterns: Prefers verbose over correct code
‚Ä¢ Training Cost: Requires thousands of human annotations
‚Ä¢ Domain Mismatch: Math PRMs don't suit programming

‚úÖ ORPS Solution

‚Ä¢ Executable Verification: Objective test results
‚Ä¢ Self-Critique: LLM explains its reasoning
‚Ä¢ Inference-Only: No training required
‚Ä¢ Code-Aware: Understands programming logic

Why Current Approaches Fail Complex Code Generation

Traditional supervision methods miss the key insight: code can be executed for immediate, verifiable feedback

Trapped in Local Optima

Repair-focused methods incrementally fix bugs but miss opportunities for superior algorithmic strategies

Expensive PRM Training

Process Reward Models require costly human annotations and suffer from reward hallucination

Sparse Feedback

Outcome-only supervision provides no guidance on how to improve or what algorithmic approach to take

The Solution: Outcome-Refining Process Supervision

ORPS unifies process and outcome supervision by leveraging executable verification. It treats the reasoning about outcome refinement as the process to supervise, eliminating the need for costly reward model training.

üí°

1. Key Insight: Execute & Reason

Unlike traditional approaches that rely on costly Process Reward Models (PRMs), ORPS leverages a fundamental insight: code can be executed for immediate, verifiable feedback.

By combining executable verification with LLMs' intrinsic self-critique capabilities, ORPS creates a powerful feedback loop where reasoning chains and code implementations co-evolve.

This inference-only approach eliminates the need for specially trained PRMs while providing superior guidance for code generation.

üîÑ

2. Three-Phase Process

1

Candidate Generation

Generate diverse reasoning and code implementations through strategic thinking

2

Execution & Profiling

Execute code and measure correctness, efficiency, and quality metrics

3

Self-Critique & Scoring

LLM provides reasoning-based critique and numerical process rewards

Tree-Structured Strategic Exploration

Unlike linear approaches, ORPS maintains multiple reasoning trajectories simultaneously, enabling exploration of different algorithmic strategies until empirical feedback identifies the superior solution.

Parallel Trajectories

Maintain multiple solution paths simultaneously (e.g., brute-force vs dynamic programming)

Execution Feedback

Concrete test results and performance metrics guide strategic decisions

Strategic Selection

Beam search with unified process and outcome rewards selects optimal paths

Basic Usage

Run ORPS on a single coding problem

# Run ORPS on a single problem
python main.py \
    --problem "Write a function to find the longest increasing subsequence" \
    --model "qwen-7b" \
    --beam-size 3 \
    --max-steps 5 \
    --candidates 2

# The framework will:
# 1. Generate multiple reasoning chains and code solutions
# 2. Execute and profile each solution
# 3. Use self-critique to guide the search
# 4. Return the best solution found

Complex Principles

RewardAnything excels when provided with clear, structured principles, especially for nuanced tasks involving multiple, potentially conflicting objectives. Define criteria, assign weights (e.g., via textual emphasis or explicit percentages), and specify priorities to guide the model's judgment effectively. This allows for fine-grained control over the evaluation process.

Executable Verification
